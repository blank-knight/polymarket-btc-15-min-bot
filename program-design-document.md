## 1. 项目概述

### 1.1 市场简介

Polymarket 的 BTC 15分钟市场（`btc-updown-15m`）是一个二元预测市场：

- **每 15 分钟一个新市场**（一天 96 个市场）
- **规则极简**: BTC/USD 结束价格 ≥ 开始价格 → "Up" 赢；否则 "Down" 赢
- **结算在链上**（Polygon, USDC），秒级结算
- **流动性**: 日交易量 $23M+（crypto 类市场总量）
- **市场结构**: Event (`btc-updown-15m-{unix_timestamp}`) → Market (Up) + Market (Down)

### 1.2 为什么能赚钱

Polymarket 的 BTC 15分钟市场本质上是一个**抛硬币市场**（50/50），但因为：

1. **人类交易者有偏见**: 倾向追涨/追跌，产生定价偏差
2. **最后 10 秒决定 15-20% 的结果**: 有动量数据的人有 Edge
3. **Bot 可以高频参与**: 一天 96 个市场，复利效应
4. **市场定价不总是 50/50**: YES 价格经常在 45-55¢ 波动

**核心 Edge**: 用 BTC 实时价格动量 + 订单流分析 + 时间衰减，在价格有偏差时下注。

### 1.3 目标

- Phase 1: 模拟运行，验证策略
- Phase 2: 小资金实盘（$50-100）
- Phase 3: 规模化（$500+），自动化运行

## 2. 策略研究

### 2.1 核心策略（按优先级）

#### 策略 1: 多时间框架共振 (Multi-Timeframe Confluence) — P0
- **逻辑**: 长期趋势 + 短期动量方向一致时，Polymarket 市场大概率没完全定价
- **Layer 1 — 趋势方向** (4h/12h/24h):
  - BTC 过去 4h 跌了 >1.5% → 趋势: Down
  - BTC 过去 24h 跌了 >3% → 趋势更强: Down
- **Layer 2 — 动量确认** (当前 15 分钟内实时走势):
  - 当前市场进行到第 10 分钟，BTC 已跌 0.5% → 确认: Down
  - 短期走势和长期趋势方向一致 → 信号强化
- **Layer 3 — 定价偏差**:
  - 两层都指向 Down，但 UP 价格还是 45¢ → 偏差存在
  - → 买 DOWN
- **安全阀 (趋势是否到极限)**:
  - RSI 指标: RSI > 75 (超买) 或 < 25 (超卖) → 降仓位或不入场
  - 价格速度衰减: 最近 30 分钟跌速/涨速明显放慢 → 趋势减弱 → 不入场
  - 关键价位: 接近整数关口、前高前低、布林带边界 → 可能反弹 → 不入场
  - 本质: 只吃"趋势明确且未到极限"的中间段，不吃头不吃尾
- **入场条件**:
  - 三层都同意 → 下重注 (Kelly 75%)
  - 两层同意 → 下轻注 (Kelly 50%)
  - 一层或信号冲突 → 不入场
- **信号源**: Binance WebSocket 实时 BTC/USD 价格 (多时间框架 K线)
- **Edge 来源**: Polymarket 交易者做短期决策，忽略大趋势；长短期共振时市场定价滞后
- **关键洞察 (by 主人)**: 长期跌 + 短期 1-14 分钟已跌 2% → 最后一分钟再涨的概率极低 → 这时入手

#### 策略 2: 最后时刻狙击 (Last-Second Snipe) — P0
- **逻辑**: 结算前 30-60 秒，BTC 已经有明确方向，但市场价格还没完全调整
- **信号源**: Binance 实时价格 vs Polymarket 当前价格
- **参数**: 确定性阈值（BTC 涨幅 > 0.1% 且距离结算 < 60s）
- **Edge 来源**: 15-20% 的市场在最后 10 秒才决定，但人类反应慢
- **风险**: Gas 费 + 滑点，需要快速执行

#### 策略 3: 均值回归 (Mean Reversion) — P1
- **逻辑**: 连续 3+ 次同方向后，下一次反方向概率略高（赌徒谬误陷阱，但市场定价会过度反应）
- **信号源**: 历史上一个市场的序列
- **参数**: 连续次数阈值，回归概率提升量
- **Edge 来源**: 市场参与者的心理偏见

#### 策略 4: 订单流分析 (Order Flow) — P2
- **逻辑**: 大单买入 "Up" → 其他 Bot 跟风 → 价格短暂偏移 → 反向下注
- **信号源**: Polymarket CLOB 订单簿深度
- **参数**: 大单阈值，跟风延迟
- **Edge 来源**: 利用其他 Bot 的行为模式

#### 策略 5: 价差套利 (Spread Arbitrage) — P2
- **逻辑**: UP + DOWN 价格之和 < $1.00 → 无风险套利
- **信号源**: 实时 UP/DOWN 价格
- **参数**: 最小价差（扣除 Gas 费后仍有利润）
- **Edge 来源**: 市场做市商的定价失误
- **注意**: 高频 Bot 竞争激烈，机会转瞬即逝

### 2.2 策略组合

| 策略 | 频率 | 胜率 | 单笔利润 | 风险 |
|------|------|------|---------|------|
| 多时间框架共振 | 中 (30-50次/天) | 55-65% | 5-12% | 低 |
| 最后时刻狙击 | 高 (50-80次/天) | 60-70% | 2-5% | 中 |
| 均值回归 | 低 (5-15次/天) | 55-60% | 8-15% | 低 |
| 订单流分析 | 中 (20-40次/天) | 53-56% | 3-8% | 中 |
| 价差套利 | 极低 (0-5次/天) | 99% | 1-3% | 极低 |

```
│                             │                     │
│  ┌─────────────┐    ┌──────▼───────┐             │
│  │ Polymarket  │───▶│   Signal     │             │

```

### 3.2 数据源

| 数据源 | 用途 | 接入方式 | 延迟 |
|--------|------|---------|------|
| Binance WebSocket | BTC/USD 实时价格 | `wss://stream.binance.com/ws/btcusdt@trade` | <100ms |
| Polymarket CLOB WS | UP/DOWN token 价格 | `wss://ws-subscriptions-clob.polymarket.com/ws` | <500ms |
| Gamma API (`/markets`) | 市场发现（btc-updown-15m） | REST | 1-2s |
| Polymarket CLOB REST | 下单、查仓位 | REST | 200-500ms |

### 3.3 关键技术点

1. **双 WebSocket 并行**: Binance 价格 + Polymarket 市场价格同时监听
2. **事件驱动**: 市场切换（每15分钟）触发新一轮决策
3. **异步架构**: asyncio + 多协程，不阻塞
4. **快速下单**: 结算前 60 秒内的"最后狙击"需要 < 2 秒延迟
5. **自动市场发现**: 根据 Unix 时间戳计算当前/下一个市场 slug

### 3.4 Price to Beat 获取策略

"Price to beat" 是每个 15 分钟市场开始时的 BTC/USD 价格。Polymarket 不通过标准 API 字段暴露此值。

**两级获取策略（按优先级）：**

1. **Gamma API 尝试**: `GET /markets?slug=btc-updown-15m-{timestamp}` → 检查 description/metadata
2. **Playwright 页面解析**: 截取 `polymarket.com/event/btc-updown-15m-{timestamp}`，解析 "Price to beat" 文本
   - 每 15 分钟新市场开启时只截 1 次
   - headless 模式 + 常驻浏览器实例（避免冷启动）
   - 超时 10 秒

⚠️ **不用 Binance 开盘价替代**：Polymarket 用自己的数据源结算（可能非 Binance），两个价格会有偏差，可能导致结算时判断错误。
Binance 的作用仅限于：实时追踪 BTC 价格走势（用于动量/趋势/RSI 计算）。

### 3.5 市场发现逻辑

```
市场 slug: btc-updown-15m-{unix_timestamp}
timestamp = 当前时间向下取整到 15 分钟的 Unix 时间戳

示例:
  14:00 → btc-updown-15m-1743849600
  14:15 → btc-updown-15m-1743850500
  14:30 → btc-updown-15m-1743851400
```

## 6. 风险管理

### 6.1 交易风控

| 规则 | 值 |
|------|---|
| 单笔最大投入 | bankroll 的 5% |
| 单日最大亏损 | bankroll 的 10% |
| 连续亏损暂停 | 5 次连亏后暂停 1 小时 |
| 单市场最大仓位 | $10 |
| 最小 Edge | 3% (比天气市场低，因为频率高) |
| Gas 费阈值 | 利润 > 2× Gas 费才交易 |

### 6.2 技术风控

| 规则 | 说明 |
|------|------|
| WebSocket 断连 | 自动重连，5 秒超时 |
| 价格异常 | BTC 1秒内波动 > 1% → 暂停交易 |
| API 限流 | 遵守 Polymarket/Binance 限流规则 |
| 数据缺失 | 缺价格数据 → 跳过本轮 |

## 7. 与天气 Bot 的区别

| 维度 | 天气 Bot | BTC 15M Bot |
|------|---------|-------------|
| 市场频率 | 每天几个 | 每天 96 个 |
| 结算周期 | 天级 | 15 分钟 |
| Edge 来源 | 模型信息差 | 动量 + 速度 |
| 需要速度 | 慢 (秒级) | 快 (< 1 秒) |
| 数据源 | 天气模型 API | Binance WS |
| 复杂度 | 中 (概率分布) | 低 (二元判断) |
| 利润模式 | 少量大额 | 多量小额 |
| 适合场景 | 6 小时运行一次 | 24/7 持续运行 |

## 8. 预期收益模型

### 保守估计 (55% 胜率)
```
每天交易: 60 次 (跳过 36 个不确定的市场)
平均投入: $2/笔
胜率: 55%
平均赔率: 1.85 (买入 YES 54¢)
日利润: 60 × 0.55 × $1.85 - 60 × 0.45 × $2 = $61.05 - $54 = +$7.05
月利润: ~$210
```

### 激进估计 (60% 胜率, 含最后狙击)
```
每天交易: 80 次
平均投入: $3/笔
胜率: 60%
平均赔率: 1.80
日利润: 80 × 0.60 × $2.40 - 80 × 0.40 × $3 = $115.2 - $96 = +$19.2
月利润: ~$576
```
