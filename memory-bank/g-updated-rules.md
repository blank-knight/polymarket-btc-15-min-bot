# 生成器 G — 迭代规则

> 每次 M 更新后写入新版本

## v0.1 — 初始规则（2026-04-05）

来源: `strategy-research.md` 中的经验值

### 策略选择规则
- 主策略: 多时间框架趋势共振（每15分钟评估）
- 辅助策略: 动量确认 + Price to Beat 对比
- 增强策略: Polymarket 定价偏差套利
- 特殊策略: 最后 60 秒狙击

### 信号强度规则
- **Strong**: 三层信号方向一致（趋势+动量+定价）
- **Weak**: 两层信号方向一致
- **None**: 信号冲突 → 不交易

### 多时间框架阈值
- 4h 趋势: ±0.36%（当前值，可调 0.3-0.5%）
- 12h 趋势: ±0.8%（当前值，可调 0.8-1.2%）
- 24h 趋势: ±1.2%（当前值，可调 1.2-2.0%）

### Edge 规则
- 最小 Edge: 3%（MIN_EDGE = 0.03）
- 信号方向: Edge > 0 → UP, Edge < 0 → DOWN

### Kelly Sizing 规则
- 使用 Quarter-Kelly (f*/4)
- KELLY_FRACTION = 0.25
- 单笔最大仓位: 5% bankroll（MAX_POSITION_RATIO = 0.05）

### 安全阀规则
- RSI > 75 → 不追涨（UP 信号降权）
- RSI < 25 → 不追跌（DOWN 信号降权）
- 速度衰减: 5分钟涨速 > 15分钟 → 趋势可能衰竭
- 关键位: 整数关口 / 布林带边沿 / 前高前低 → 降权 50%

### 狙击规则
- 触发时间: 结算前 60 秒（SNIPER_TRIGGER_SECONDS = 60）
- BTC 方向 vs PTB 差距 > 0.05%
- Polymarket 价格仍有明显偏差
- 使用 Quarter-Kelly 的 50%（更保守）

### 自适应优化规则
- 胜率 > 65% → 增加 aggressiveness（降低阈值）
- 胜率 < 45% → 增加 conservatism（提高阈值）
- 交易频率过低 → 放宽 RSI 阈值
- 每 7 天重新评估

### 风控规则
- 日交易上限: 100 笔
- 日亏损上限: 10% bankroll
- 连续亏损冷却: 5 次连续亏损 → 暂停 60 分钟
- 单笔仓位: ≤ 5% bankroll
