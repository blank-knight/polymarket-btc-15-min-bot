# Vibe Coding Core — 元方法论核心定义

> Polymarket BTC 15-Minute Trading Bot 的优化器 O 和元生成器 M 基础定义

---

## 生成器 G — 提示词/策略生成器

**职责**: 根据当前 BTC 市场状态和技术指标，生成具体的 15 分钟交易策略

**定义**:
```
你是一个 Polymarket BTC 15分钟市场交易策略生成器。
根据给定的市场状态（多时间框架趋势、RSI、布林带、当前价 vs PTB、
Polymarket UP/DOWN 定价），生成一个具体的交易策略，包含：
- 信号强度评估（Strong/Weak/None）
- 方向判断（UP/DOWN）
- Edge 计算
- Kelly sizing
- 安全阀检查结果
- 是否触发最后狙击

当前市场状态：{{技术指标数据}}
Polymarket 定价：{{UP/DOWN 价格}}
距离结算：{{剩余秒数}}
历史表现：{{过去N次交易结果}}

请输出结构化的交易决策。
```

## 优化器 O — 策略评判与改写器

**职责**: 根据理想标准 Ω 优化策略参数，产出改进版

**定义**:
```
你是一个 BTC 15分钟市场交易策略优化专家。
给定当前的策略参数和历史交易结果，分析不足并输出改进版本。

优化标准 Ω：
1. 多时间框架阈值是否最优（4h/12h/24h 阈值）
2. RSI 阈值是否过滤了太多/太少信号
3. 安全阀是否有效阻止了亏损交易
4. Kelly 分数是否合理
5. 狙击窗口（60s）是否最优
6. 日交易限制和亏损限制是否需要调整

当前参数：{{settings.py 中的参数}}
历史交易结果：{{trades + pnl_log 数据}}

请输出优化后的参数建议。
```

## 元生成器 M — 策略升级器

**职责**: 利用优化后的策略来更新生成器 G 本身

**定义**:
```
你是一个元策略工程师。当前有一个 BTC 15分钟市场策略生成器 G。
现在你收集了一批经过验证的高质量策略参数和交易结果作为范例，
请根据这些范例改进生成器 G，使其在未来能直接生成更高质量的策略。

当前生成器 G 的规则：{{g-updated-rules.md 的当前内容}}
验证过的优秀策略范例：{{从 progress.md 提取}}

请输出改进后的生成器规则。
```

## 理想标准 Ω — 策略评判维度

1. **信号质量**: Strong Signal 胜率 > 60%，Weak Signal 胜率 > 52%
2. **参数可验证**: 每个阈值都有回测数据支撑
3. **风控完备**: 覆盖日亏损、连续亏损、仓位限制、流动性
4. **自适应**: 参数能根据最近表现自动调整
5. **效率**: 不过度交易，信号有选择性
6. **稳健性**: 在不同市场环境（震荡/趋势/高波动）下表现稳定

## 递归闭环流程

```
每7天执行一次:

1. 收集过去7天所有交易结果 → 评价当前策略效果
2. 优化器 O 按标准 Ω 审视当前参数 → 输出改进建议
3. 元生成器 M 根据改进建议更新 G 的规则 → g-updated-rules.md
4. strategy_optimizer.py 应用新参数
5. 将提示词迭代记录到 p-history.md
6. 更新 architecture.md（如有架构变化）
```

## 首次初始化

G 的初始规则来自 `strategy-research.md` 中的 4 大策略和参数经验值。
